Welcome to the Fizzbuzz challenge!

Fizzbuzzz is a coding challenge given to prospective programmers for years and has become famous (or should I say 'infamous') because of how many people fail this test miserably. Some chalk it up to nerves, some just don't know how to approach the problem but most people just get thrown off guard.

In reality, the challenge is super easy to accomplish. You just need to keep calm and remember the fundamentals you learned in school. Here we will show you how to break down the problem:


Ok, so the way this works is you need to print out the numbers 1 through 100. However, you need print 'Fizz' if the number is divisible by 3, 'Buzz' if the number is divisible by 5, and 'FizzBuzz' if the number is divisible by both 3 and 5.

Let's start by setting up a for loop to iterate through the numbers :

for (var i = 1; 1 < 101; i++)

Great! Now, lets be sure we can print the results out :

for (var i = 1; 1 < 101; i++){console.log(i)}

Ok now the basic structure looks good but this won't do to suit our needs. We are going to need to meet some conditionals if this solution is ever going to get off the ground. In a problem like this, I like to look for commonalities with the conditions and the first thing that stands out in my mind is that 3 and 5 are both divisible by 15.

So, lets use this number with modulo to discover which integer gets assigned the FizzBuzz' value in our loop. We can apply this using an if condition inside our for loop (not forgetting to leave the number unchanged if it does not meet our condition) like so:

for (var i = 1; i < 101; i++){if ((i % 15) === 0) {console.log('FizzBuzz');} else {console.log(i);}}

With this, we can see that if there is no remainder, 'FizzBuzz' is applied. Cool, right?


Great! So now we are approaching the home stretch. Now that our function is filtering out our dual condition, all we need to do is apply the same technique to find each condition individually. Lets add two more else ifstatements in order to assign 'Fizz' and 'Buzz' to whichever integers happen to meet one codition or the other :

for (var i = 1; i < 101; i++){ if ((i % 15) === 0) { console.log('FizzBuzz'); } else if ((i % 3) === 0){ console.log('Fizz'); } else if ((i % 5) === 0){ console.log('Buzz'); } else { console.log(i); } }
